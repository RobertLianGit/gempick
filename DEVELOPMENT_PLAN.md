# 「心跳之间」核心开发文档

最后更新：2026-07-19

状态：MVP 已实现，可本地运行并静态构建。本文描述当前代码行为，不再是早期提案。

## 1. 产品定位

产品名：**心跳之间**

SEO 标题：**心跳之间｜邓紫棋歌曲个人Pick体验**

首页核心文案：

- 你最Pick的G.E.M.邓紫棋歌曲；
- 找到你最后想留下的那一首，了解每一首歌曲背后的感觉；
- 开始选择。

这是一段有情绪的个人歌曲选择体验，不是客观评选、公开投票或体育赛事。用户从愿意带上的作品开始，在一次次两首歌的相遇中做出选择，最后得到“我最 Pick 的 G.E.M. 邓紫棋的歌”和完整选择旅程。

产品价值包括两部分：

1. 帮助用户得到一个只属于当下的个人结果；
2. 在选择过程中重新认识每首歌的主题、发行背景和版本信息，并鼓励用户在不熟悉时去正规音乐平台试听。

## 2. 项目边界

### 当前必须保持

- 非官方、非商业粉丝项目；
- 与艺人本人、工作室、唱片公司和相关团队不存在隶属或授权关系；
- 不托管音频，不展示歌词、MV、专辑封面、艺人照片、演唱会照片或官方 Logo；
- 不建立账号，不收集姓名、邮箱、手机号或精确位置；
- 不生成全站排名，不使用“最强歌曲”“第一名”“正确答案”等权威暗示；
- 作品事实和编辑摘要保留来源链接；
- 外部试听只跳转至正规音乐平台，不嵌入或抓取播放器。

### 当前不做

- 登录、注册、用户资料和云端同步；
- 评论、投稿、图片上传和社交互动；
- 广告、打赏、会员、票务、周边和赞助；
- 用户总体排行榜；
- 复杂后台和公开结果链接；
- 未取得授权的封面、音频、歌词或肖像素材。

## 4. 当前用户流程

### `/` 首页

首页负责让用户在首屏理解产品，并在开始前看到最终结果会是什么样：

- 品牌“心跳之间”；
- 确认版核心文案与“开始选择”按钮；
- 三步说明：带上专辑 → 让歌曲相遇 → 留下最 Pick 的一首；
- 15 张发行、129 个专辑录音版本，并纳入精选、影视、公益与合作单曲索引；
- 非官方声明；
- “最后会得到什么”结果预览卡；
- 结果预览显示“我最 Pick 的 G.E.M. 邓紫棋的歌”、64 首示例、63 次选择和旅程节点，并明确标注为非真实结果。

### `/select` 轻量选歌

页面标题：**先留下你想带上的歌**。

行为：

- 首次进入自动留下固定的“起点十首”；
- 起点十首为：`回忆的沙漏`、`A.I.N.Y. 爱你`、`泡沫`、`多远都要在一起`、`来自天堂的魔鬼`、`光年之外`、`倒数`、`句号`、`新的心跳`、`超能力`；
- 起点十首不是直接开赛，而是降低第一次选择的压力，用户可以随时放下或换歌；
- 专辑卡显示年份、发行类型、歌曲数量和抽象轨道；
- 用户可以逐首加入，也可以整张加入或撤下；
- 可展开专辑曲目并直接逐首操作；
- 页面顶部显示已选数量、还差多少、预计选择次数和自动保存状态；
- “继续选择”按钮位于内容上方，不要求用户滚动到底部；
- 顶部提供“重新开始一段选择”，确认后清除当前歌曲、相遇顺序和进度，并回到起点十首；
- 超过 128 首时阻止继续添加并给出提示；
- 少于 16 首时继续停留在选歌页，达到 16 首即可进入相遇页。

### `/wildcards` 逐首挑选兼容入口

主流程已经把逐首挑选并入 `/select`；该页面只保留给旧书签或直接访问的用户。

行为：

- 顶部显示当前数量、剩余数量、自动保存状态和“开始相遇”按钮；
- 顶部同样提供“重新开始一段选择”，确认后返回轻量选歌页；
- 按钮位于候选内容上方；
- 候选歌曲是紧凑方格矩阵，而不是两列长列表；
- 桌面每行 8 个方格，平板 6 个，手机 3 个；
- 每个方格显示歌曲名、专辑、年份和“加入这首/先放下”；
- 支持按歌曲名和专辑名搜索；
- 达到 16 首前“开始相遇”不可用。

### `/bracket` 这次相遇

页面标题：**让歌曲开始相遇**。

显示：

- 这次留下的歌曲数量；
- 预选对决数量；
- 随后登场的正式规模；
- 总选择次数；
- “生成这次相遇”按钮；
- `GEM-{seed}` 格式的旅程编号；
- 第一段相遇预览和“继续选择”按钮。

生成结果后必须稳定。刷新只读取本机已保存结果，不重新洗牌。

### `/play` 歌曲相遇

核心标题：**这一轮，你更想留下哪一首？**

每一边必须保持完全对称，并同时展示：

- 歌曲名、专辑和发行年份；
- 自行整理的“作品线索”；
- 可展开的专辑发行背景；
- 语言、唱片信息和资料来源；
- “留下这首”按钮；
- 明显的“去音乐平台听听”入口。

试听引导：

- 页面顶部提示“先去听听，再回来选择”；
- 第一次点击任意试听入口时询问常用平台，可选 QQ 音乐、Apple Music、网易云音乐和酷狗音乐；
- 平台偏好只保存在当前浏览器，之后对决页、选择反馈页和结果页共同沿用；
- 全站页脚允许随时更改平台；
- 外链在新标签页打开；
- 选择 Apple Music 时，如果发行来源中有精确发行页则优先打开，否则进入歌曲搜索页；
- 其他平台进入包含“邓紫棋 + 歌名”的官方搜索页；
- 明确告诉用户打开外链不会丢失进度；
- 不嵌入播放器，不自动播放。

选择后显示：

- “它会继续留下”；
- 暂时告别的另一首歌；
- 歌曲主题摘要；
- 语言和发行背景；
- 继续选择或回到刚才的选择。

辅助操作：

- 左右方向键选择两边歌曲；
- 撤销最近一次选择；
- 暂停并看看完整旅程；
- 进度文案显示已完成次数和总次数。

### `/result` 结果页

核心文案：

- 我最 Pick 的 G.E.M. 邓紫棋的歌；
- 最后留下的，是这一首；
- 这不是哪首歌“最好”，只是这一刻，你最想留下它。

结果包含：

- 最后留下的歌曲、专辑和年份；
- 最初歌曲数量；
- 总选择次数；
- 最后一次遇见；
- “它走过的路”；
- 可展开的完整旅程；
- “去听这首歌”，直接使用已选择的听歌平台；
- “生成结果图片”，在浏览器内生成 4:3 PNG 并显示保存预览；
- “重新开始一段选择”。
- 完整个人排行榜：按歌曲停止继续留下的阶段形成第 1 名、第 2 名、第 3–4 名、第 5–8 名等并列层级，并覆盖全部参与歌曲。

排行榜不做伪精确排序。同一阶段没有发生彼此比较的歌曲必须显示为并列；如果未来需要严格的 1–N 名次，必须另加排序比较流程，不能从当前单败选择中虚构。

结果图片格式：

```text
我最 Pick 的 G.E.M. 邓紫棋的歌
{歌曲名}
{专辑名} · {年份}
这是这一刻，我最想留下的那一首。
{歌曲数 / 选择次数 / 继续留下次数}
心跳之间｜非官方粉丝项目
```

结果分享不得依赖 Clipboard API。局域网 HTTP 地址在 Safari 中不属于安全上下文，剪贴板写入会失败；PNG 必须通过 Canvas 在本机生成。

### `/about`、`/legal` 与错误页

- `/about` 解释玩法、预选对决、自动保存、作品背景和试听入口；
- `/legal` 标题为“关于这个非官方发起的粉丝项目”，法律正文保持正式清楚；
- `app/error.tsx` 使用“这片星轨暂时没有连上”和“再试一次”；
- 发布前必须配置真实版权联系邮箱，不得虚构。

## 5. 内容与运行时数据模型

源数据：`data/gem-discography.json`。专辑录音版本与 `standalone_and_featured_releases` 中的精选、影视、公益和合作单曲都会进入候选池。

运行时目录转换在 `src/data/catalog.ts` 完成。

```ts
type Track = {
  id: string;                 // release-id--position 或 single--year--index
  title: string;
  titleEn?: string;
  albumId: string;
  albumTitle: string;
  releaseYear: number;
  language: string[];
  summary: string;            // 自行整理的作品线索
  albumIntroduction: string;  // 专辑发行背景
  label?: string;
  version?: string;
  sourceUrls: string[];
};
```

不同语言、现场、混音和重录版本使用不同 ID。创建旅程前只按录音版本 ID 去重，不按歌曲标题去重。

## 6. 相遇算法

内部仍采用单败淘汰树，但用户界面只使用音乐旅程语言。

令 `N` 为去重后的歌曲数量，`S` 为不大于 `N` 的最大 2 的幂：

```text
N = 16  → S = 16，预选对决 0 次
N = 17  → S = 16，预选对决 1 次
N = 32  → S = 32，预选对决 0 次
N = 43  → S = 32，预选对决 11 次
N = 64  → S = 64，预选对决 0 次
N = 100 → S = 64，预选对决 36 次
N = 128 → S = 128，预选对决 0 次
```

预选数量为 `N - S`。预选胜者与稍后登场的歌曲共同组成 `S` 首正式顺序。总选择次数始终为 `N - 1`。

算法要求：

1. 用随机种子初始化可复现伪随机数；
2. 使用 Fisher–Yates 洗牌；
3. 同一 `seed + selectedTrackIds` 必须生成同一结构；
4. 生成后保存完整结构，不在恢复时重新计算；
5. 预选位置和稍后登场位置都由同一个种子决定；
6. 不设置热门保护或同专辑回避；
7. 16–128 首必须无重复、无遗漏并且可以结束。

## 7. 状态与本机保存

localStorage key：

```text
gempick:lastsong:v1
```

听歌平台偏好使用独立 key：

```text
gempick:music-platform:v1
```

虽然产品品牌已经更名，MVP 暂时保留原 key，避免升级时丢失已有本机进度。只有明确设计迁移逻辑后才能更改。

保存：

- 已选歌曲 ID；
- 随机种子和旅程编号；
- 全部相遇节点；
- 每次选择；
- 当前进度和更新时间。

不保存：

- 姓名、邮箱、手机号；
- 精确位置；
- 云端个人结果；
- 未经同意的分析数据。

重新开始必须显示：

> 重新开始后，之前的选择会被清除。确定要从头来吗？

## 8. 技术架构

- Next.js 16 App Router；
- React 19；
- TypeScript 严格模式；
- 自定义全局 CSS，不使用 Tailwind；
- JSON 内容数据；
- React 本地状态 + localStorage；

## 10. MVP 验收标准

新用户应当能够：

1. 在首页理解这是个人选择体验，并看到最终结果示例；
2. 明确知道项目非官方；
3. 从起点十首开始，通过逐首挑选或整张专辑加入留下 16–128 首歌；
4. 在内容上方随时看到已选数量、保存状态和下一步按钮；
5. 在选歌过多时通过顶部入口确认并重新开始；
6. 生成稳定不变的相遇顺序；
7. 在每次选择前理解两首歌的作品线索和发行背景；
8. 不熟悉时前往音乐平台试听并安全返回；
9. 刷新后继续刚才的选择；
10. 撤销最近一次选择；
11. 得到唯一的“我最 Pick 的 G.E.M. 邓紫棋的歌”；
12. 直接前往自己选择的音乐平台试听结果歌曲；
13. 查看它走过的路和完整旅程；
14. 查看覆盖全部参与歌曲、同阶段明确并列的个人排行榜；
15. 生成不含封面、歌词、Logo 或官方排名暗示的结果图片；
16. 在手机上完成平台选择、试听跳转、排行榜浏览、结果图片预览与保存。
