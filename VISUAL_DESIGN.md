# 「心跳之间」视觉与交互规范

最后更新：2026-07-19

实现位置：`app/globals.css`

## 1. 视觉方向

主视觉：**紫色星轨 / Purple Orbit**。

核心情绪：

> 在属于自己的音乐宇宙里，让歌曲一次次相遇，理解它们背后的感觉，留下此刻最 Pick 的那一首。

关键词：浪漫、私密、星空、心跳、相遇、留下、旅程、余韵。

页面应像粉丝与歌曲相处的空间，而不是体育赛事、投票平台、游戏广告或艺人官方网站。

## 2. 视觉原则

- 情绪优先，但信息必须清楚；
- 两首歌始终保持相同尺寸、亮度、信息量和按钮层级；
- 作品背景与试听入口在两边完全对称；
- “加入、相遇、继续留下、抵达”有递进的仪式感；
- 下一步操作优先放在内容上方，长内容只作为参考；
- 所有装饰使用原创抽象图形；
- 不使用真人照片、专辑封面、官方 Logo、签名、舞台或 AI 肖像；
- 动效关闭后仍能完整使用。

## 3. 色彩系统

```css
--orbit-night: #0B0A25;   /* 页面背景 */
--orbit-deep: #15113A;    /* 深层背景 */
--orbit-plum: #30205A;    /* 卡片与导航 */
--orbit-violet: #8C62D9;  /* 主交互 */
--orbit-lilac: #C6A7F3;   /* 高亮与轨道 */
--orbit-rose: #E7A9C0;    /* 情绪强调 */
--orbit-gold: #F5D99B;    /* 最终结果与旅程节点 */
--orbit-ivory: #FFF7EA;   /* 主文字 */
--orbit-muted: #B7AFCA;   /* 次级文字 */
```

规则：

- 背景使用深蓝紫渐变，不使用纯黑；
- 主按钮使用紫色到玫瑰色渐变；
- 金色主要用于最终结果、继续留下和重要旅程节点；
- 两张歌曲卡不使用不同主色暗示选择；
- 所有正文满足可读对比度，不能为氛围牺牲清晰度。

## 4. 背景与图形

背景由深色宇宙渐变、淡同心轨道、少量固定星点和柔光组成。

允许：

- 轨道椭圆；
- 小型星点；
- 半透明圆形和月相；
- 细线连接；
- 轻微模糊和颗粒感。

避免：

- 星云照片；
- 大量循环闪烁；
- 麦克风、舞台和人物剪影；
- 模仿官方专辑或演唱会视觉；
- 需要授权的图片占位。

## 5. 字体与排版

- 中文大标题：宋体或高质量衬线字体；
- 正文和操作：清晰无衬线字体；
- 数字、编号和进度：等宽字体；
- 不使用模仿艺人签名的字体。

当前层级：

```text
首页 Hero       52–94px
页面标题        40–70px
结果歌曲名      64–132px
对决歌曲名      26–38px
正文            14–17px
辅助信息        9–13px
```

长歌名最多显示两行；单曲矩阵使用截断和原生 `title` 提示，避免撑破方格。

## 6. 核心组件

### 顶部操作面板

用于专辑选择和单曲补选页：

- 位于页面说明下方、长内容上方；
- 同时显示已选数量、剩余数量或预计选择次数；
- 明确显示“这段选择已经替你保存好了”；
- 包含当前页面的下一步主按钮；
- 提供低层级的“重新开始一段选择”入口，并在清除前要求确认；
- 不把唯一 CTA 放在长列表底部；
- 移动端纵向堆叠，按钮占满可用宽度。

### 专辑卡

- 三列桌面网格、两列平板、单列手机；
- 半透明深紫背景、1px 边框、24px 圆角；
- 显示年份、发行类型、歌曲数量和抽象轨道；
- 选中后使用金色星点和更亮边框；
- 提供“整张加入”“撤下整张专辑”和展开歌曲入口。

### 单曲补选方格

- 桌面每行 8 个；
- 平板每行 6 个；
- 手机每行 3 个；
- 方格保持接近 1:1；
- 显示歌曲名、专辑、年份和“加入这首/先放下”；
- 选中后使用金色边框和星点；
- 方格矩阵只作为参考内容，“开始相遇”按钮位于上方。

### 歌曲相遇卡

- 桌面两张卡左右排列，中间以细轨道和 VS 连接；
- 手机两张卡上下排列；
- 桌面最小高度约 680px，以容纳作品背景和试听入口；
- 两边显示完全相同的信息结构：
  - 年份与专辑；
  - 歌曲名和英文名；
  - 作品线索；
  - 可展开的发行背景；
  - 语言、唱片信息和资料来源；
  - “留下这首”；
  - “去音乐平台听听”。
- 试听入口使用独立描边面板，不再是容易忽略的小字链接；
- 移动端仍然保留试听入口，不隐藏。

### 进度轨道

- 使用细线和发光星点，不使用厚重游戏进度条；
- 同时显示当前相遇和“你已经做出了 N 次选择”；
- 进度星点沿轨道移动；
- 数字使用等宽字体。

### 非官方声明

确认文案：

> 非官方粉丝项目｜与艺人本人及相关团队不存在隶属或授权关系

出现位置：

- 首页首屏；
- 全站页脚；
- 最终结果区域。

样式保持低饱和、可信和清楚，不使用强警告框压过主情绪。

## 7. 首页结果预览

首页必须在三步说明之后展示最终结果效果，让用户在开始前理解产品产出。

预览卡内容：

- “效果示例 · 非真实结果”标签；
- “我最 Pick 的 G.E.M. 邓紫棋的歌”；
- 示例歌曲占位“你的那一首”；
- “从 64 首歌曲开始 · 经过 63 次选择”；
- 最初相遇、最后八首、最后留下的旅程节点；
- “最后留下的，是这一首。”

视觉：

- 金色结果文字；
- 大型椭圆轨道；
- 深紫半透明卡片；
- 不使用真实封面或照片，避免让示例看起来像用户真实结果。

## 8. 作品背景与试听引导

作品理解是核心体验，不是赛后附属信息。

每次相遇前：

- 两张卡都直接显示作品主题摘要；
- 用户可展开专辑发行背景；
- 提供语言、唱片信息和资料来源；
- 页面上方提示“先去听听，再回来选择”；
- 告知打开外链不会丢失进度。

试听按钮：

- 使用音符图标、描边容器和两级文字；
- 主文案“去音乐平台听听”；
- 辅助文案“不太熟悉？先去听听，再回来选择；进度还在”；
- 在新标签打开；
- 不使用自动播放和站内嵌入。

首次试听的平台选择层：

- 桌面为居中的 2×2 平台卡片；
- 手机为底部出现的单列选择面板；
- 明确提示偏好只保存在当前浏览器；
- 页脚持续提供更改平台入口；
- 不读取音乐账号，不伪装为平台授权登录。

选择后：

- 再次展示“歌曲背后的感觉”；
- 补充专辑发行背景；
- 用“它会继续留下”和“暂时告别”描述结果。

## 9. 页面动效

### 首页

- 标题和轨道淡入；
- 首次进入时总时长约 800–1100ms；
- 主按钮尽快可点击；
- 背景轨道只做低存在感运动。

### 专辑选择

- 卡片进入时长 280ms；
- 每张错峰 18ms；
- 15 张卡全部出现约 530ms；
- hover 上浮 4px；
- 选中反馈约 240ms；
- 首次按需编译时显示骨架屏，不出现无反馈空白。

### 单曲补选

- 方格点击立即改变颜色、边框和“加入这首/先放下”；
- 不使用歌曲飞入长列表的复杂布局动画；
- 达到 32 首后只强化顶部按钮，不使用烟花或爆炸效果。

### 生成这次相遇

- 轨道与星点强化仪式感；
- 结果数据必须先固定，再展示动画；
- 刷新后不重复制造“重新抽签”的错觉。

### 歌曲相遇

- 左右卡片对称进入，约 450ms；
- hover 反馈约 180ms；
- 选择后进入独立反馈页；
- 两边不能因为动画强度不同而暗示答案。

### 最终结果

- 金色仅在抵达结果时成为主视觉；
- 轨道和星点一次性出现；
- 不使用持续撒花；
- 完整旅程可以横向滚动查看。
- 三个主要动作按“去听这首歌 → 生成结果图片 → 看看完整旅程”排列；
- 结果图片以 4:3 比例预览，桌面居中、手机从底部出现；
- 手机支持长按图片或点击“保存图片”。
- 结果统计之后展示个人排行榜；前八名直接展开，后续并列层级使用折叠面板；
- 排行榜桌面四列、平板两列、手机单列；同层歌曲保持完全相同的视觉权重。

## 10. 响应式规则

### 大于 900px

- 首页结果预览为左右两栏；
- 专辑三列；
- 单曲补选八列；
- 两首歌曲左右相遇。

### 681–900px

- 专辑两列；
- 单曲补选六列；
- 首页结果预览改为单列。

### 680px 及以下

- 页面左右边距 14px；
- 固定 `text-size-adjust: 100%`，避免 Safari 自动放大局部正文；
- 首页主标题控制在 42–54px，普通流程页主标题约 34px；
- 歌曲相遇页主标题 24px、歌曲名 24px、作品线索正文 13px；
- 辅助说明与元信息使用 8–10px，保证层级清楚但不抢占首屏；
- 主要按钮字体 13px，触控高度仍不低于 46px；
- 专辑单列；
- 单曲补选三列；
- 顶部操作面板纵向排列；
- 两张歌曲卡上下排列；
- 选择按钮和试听入口占满宽度；
- 结果操作按钮纵向排列。
- 听歌平台选择改为单列，结果图片预览不超出可视高度。

## 11. 无障碍与性能

- 支持 `prefers-reduced-motion`；
- 所有按钮有可见焦点；
- 单曲方格使用 `aria-pressed`；
- 外链标明新窗口方向符号；
- 两张卡信息量和顺序一致；
- 页面骨架只表达加载，不伪造真实内容；
- 动画优先使用 `transform` 和 `opacity`；
- 移动端减少模糊和大面积阴影；
- 开发模式预热后页面响应目标低于 50ms；
- 静态生产构建不依赖运行时数据请求。

## 12. 视觉验收标准

- 首页首屏能识别“心跳之间”和 Pick 体验；
- 不开始操作也能在首页看到最终结果示例；
- 选择页的数量、保存状态和下一步按钮不需要滚到底部寻找；
- 全部候选录音版本以矩阵快速扫描，不形成两列超长列表；
- 对决页能在选择前了解两首歌的背景；
- 不熟悉歌曲时，试听入口足够明显；
- 双方卡片绝对对称；
- 手机端仍保留作品背景、试听引导和 Purple Orbit 氛围；
- 不依赖封面、照片、音频、歌词或 Logo 也能成立；
- 动效关闭后信息和流程完整。
